Getting Started

This documentation covers the architecture, setup, and maintenance of the Rashid platform. The platform is built as a static site deployed on Vercel with dynamic features powered by Supabase.

Tech Stack:

  • Frontend: HTML5, CSS3, Vanilla JavaScript with Three.js for 3D
  • Backend: Supabase (PostgreSQL, Auth, Realtime, Storage)
  • AI: Google Gemini API, OpenRouter
  • Deployment: Vercel with GitHub CI/CD
  • PWA: Service Worker with offline caching

Project Structure

The repository is organized as follows:

  • index.html — Main landing page with hero, models, vault, projects, about, and contact sections
  • css/ — Stylesheets (style.css, admin.css, app-mode.css)
  • js/ — JavaScript modules (script.js, admin.js, rashid-ai-v2.js, supabase-config.js, etc.)
  • games/ — Game applications (farm-game, rust-game, arcade hub)
  • apps/ — Web applications (quiz-app, calculator-vault, quran-app)
  • models/ — AI model interfaces (Rashid-app)
  • vault/ — Knowledge repository (prompts, code, archive, media, docs, api)
  • admin/ — Admin panel (login, dashboard)

PWA Configuration

The site is configured as a Progressive Web App with offline support.

Service Worker: The sw.js file handles caching strategies for static assets, API responses, and offline fallback. The offline page at offline.html is displayed when the user has no network connection.

Manifest: The manifest.json defines app name, icons, theme color, and display mode for installable PWA experience.

Deployment Checklist

Before deploying to production, verify the following:

  • All environment variables are configured in Vercel (Supabase URL, anon key, Gemini API key)
  • Supabase RLS policies are active on all tables
  • Sitemap and robots.txt are present and correctly reference the live domain
  • Canonical URLs point to https://rashid-wep.vercel.app/
  • All JavaScript files are cache-busted with version query strings
  • Service worker is registered and caches critical assets
  • SEO meta tags are present on all public pages

Maintenance

Regular maintenance tasks include:

  • Reviewing Supabase database performance and query optimization
  • Updating dependencies (Three.js, Supabase JS, Font Awesome)
  • Checking Vercel deployment logs for build errors
  • Testing PWA offline functionality after changes
  • Verifying Google AdSense ads are displaying correctly
