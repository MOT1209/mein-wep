Portfolio Optimization: From 72 to 85 in One Day

I spent a full day optimizing my portfolio site (Rashid Command Center) and brought it from a 72/100 to an 85/100. Here are the key improvements.

Results at a Glance

Score Points Gained

1. Image Optimization (WebP)

Converted all images to WebP format with quality=80:

  • logo.png 96KB → logo.webp 11KB (88% smaller)
  • profile.jpg 163KB → profile.webp 63KB (61% smaller)
  • avatar.jpg 66KB → avatar.webp 23KB (65% smaller)

Total: 325KB → 97KB (70% reduction)

2. SEO & Structured Data

Added comprehensive JSON-LD schemas:

  • WebSite schema with search action
  • Person schema with social links
  • BreadcrumbList schema for navigation
  • hreflang tags for English, Arabic, German
  • RSS feed for the blog
  • Dual theme-color meta tags

3. Accessibility

Major accessibility improvements:

  • 53 icons with aria-hidden="true"
  • Focus trap in settings modal
  • Focus-visible styles for keyboard navigation
  • Skip link for screen readers
  • Proper aria-label on all sections

4. Performance

  • Performance Mode stops Three.js animation loop (CPU/GPU savings)
  • Deferred non-critical CSS
  • Direct module imports instead of script redirect
  • Reduced CSS file duplication

5. Security

  • rel="noopener noreferrer" on all external links
  • Moved API key to Cloudflare Worker secret
  • Proper Content Security Policy headers

Key Takeaway

You don't need a complete redesign to go from good to great. A systematic audit of SEO, performance, accessibility, and security can yield massive improvements in a single day. The most impactful changes were:

  1. Image compression (WebP) — instant performance gain
  2. Structured data — better search engine understanding
  3. Accessibility fixes — inclusive design for all users
  4. GitHub integration — live API data shows real activity
